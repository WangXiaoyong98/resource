1、详细介绍一下 CPU 三级缓存（L3 Cache）
    三级混存是 CPU 和主内存 （RAM）之间的最后一道高速缓存，在现代多核 CPU 中，三级缓存通常是一个共享的、大内存的缓存层
    位于核心私有的L1\L2缓存和系统主内存之间

    核心目标是：降低CPU访问内存的频率和延迟，提高CPU的性能和效率

    L3 缓存的核心特性 
        1、共享性（这个是L3缓存和L1 L2 缓存的最本质的区别）
            私有缓存的问题： 在L1、L2缓存中，每个核心的数据都是独立的， 如果 core0 需要访问 core1 刚处理过的数据，以前需要通过缓存一致性协议在核心间传递，或者从内存重新读取，效率较低

2、介绍一下DMA   具体原理介绍一下

直接寄存器存储
    核心组件：
        DMA控制器：负责DMA传输的硬件模块
        通道：每个DMA控制器有多个独立的通道，可以同事服务多个外设
        寄存器组：源地址、目标地址、传输计数、控制状态等

    1. 初始化阶段（CPU负责）
   ├─ 设置源地址（外设数据寄存器地址或内存地址）
   ├─ 设置目标地址（内存地址或外设数据寄存器地址）
   ├─ 设置传输数据量（字节数）
   └─ 设置传输模式（单次/周期/请求等）

    2. 传输阶段（DMA控制器负责）
       ├─ 向总线仲裁器申请总线控制权
       ├─ 获得总线控制权后开始传输
       ├─ 每传输一个数据单元，地址指针递增，计数器递减
       └─ 计数器为0时，传输结束

    3. 结束阶段
       └─ 发送中断信号通知CPU传输完成 

    传输模式：
        1、单字节模式： 每次传输一个字节、传输后立即释放总线控制权，CPU和外设交替使用总线，适合低速设备
        2、块传输模式： 一次性传输整个大数据，传输期间一直占用总线，会阻塞CPU
        3、请求传输模式： 

DMA占用总线传输数据的时候，CPU在做什么
    执行其他应用程序的指令
    处理其他外设的中断
    进行运算或者参与调度
    唯一不能做的就是访问 DMA 正在使用的总线

总线冲突时候的等待： 当CPU和DMA需要访问同一条总线时，
    1、CPU会进入总线等待状态
    2、流水线暂停
    3、CPU内部寄存器状态保持不变
    


3、介绍一下虚拟地址到物理地址的转换
    虚拟地址目的：让每个进程都觉得自己有一块独立的内存空间
    实现手段：虚拟地址转换是由MMU硬件主导，在操作系统协助下完成的一个透明过程。它通过页表建立映射，利用TLB加速，并为每个进程提供了独立、安全、连续的内存抽象。


4、ARM V7 有多少个寄存器，有哪些模式，异常等级有几个

    EL0 无特权
    EL1 内核模式
    EL2 虚拟化
    EL3 安全监控

    R13 是栈指针
    R14 链接寄存器
    R15 PC 指向当前正在取指的指令地址。在ARM状态下，读取pc得到当前指令+8；写入pc会触发跳转。

    CPSR（current Program Status Register ） : 程序状态寄存器

    SPSR（Saved Program Status Register）
    每种异常模式都有一个独立的SPSR，用于在异常发生时保存被中断程序的CPSR，以便返回时恢复。


5、obj -m 和 obj -y 的联系与区别
obj-m 和 obj-y 是 Linux 内核构建系统（Kbuild）中用于控制内核模块编译的核心变量。它们都定义在内核的 Makefile 中，指示哪些源文件需要编译，以及以何种方式编译。
