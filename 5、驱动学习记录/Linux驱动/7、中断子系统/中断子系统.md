1、按键、定时器、网络数据等信号会把中断信号给控制器，控制器再将中断信号给CPU

2、中断的处理流程

外设中断信号给中断控制器，中断控制器将中断信号给CPU，如果是对于共享中断，则会进入对应共享中断的处理函数，处理函数中会对中断来源有一个判断

3、对于linux子系统中最重要的函数是 request_irq(unsigned int irq, irq_handler_t handler, unsigned int flags, const char *name，void *dev)

    interrupt_parent = <&gpio1>

5、对于多core的芯片来说
    PPI（Private Peripheral Interrupt）
    SPI（Shared Peripheral Interrupt）
    SGI（software generated interrupt）
    GIC（Generic Interrupt Controller）

6、中断中还会涉及到 休眠-唤醒 POLL机制、异步通知、定时器、中断的线程化处理

多core的核间通信的话可以使用中断来处理

7、中断可以处于多种不同的状态
    1、非活动状态：中断没有被触发
    2、挂起状态：中断被触发，但是没有被处理
    3、活动状态：中断被触发，且正在处理中

8、中断的发生、处理过程
    CPU执行完当前指令、检查到发生了中断，跳转到向量表
    保存现场，执行GIC 提供的处理函数，恢复现场


9、异常向量表的安装
    在裸机程序中，通常是用的 0 地址
    在linux中，通常是用的 0xffff0000 地址,这个地址是一个虚拟地址
