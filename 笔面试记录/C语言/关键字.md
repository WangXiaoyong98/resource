# 关键字 
## 预处理和关键字
### 宏定义是在编译的哪个阶段被处理的
> 宏定义是在编译的预处理阶段被处理的
> 预处理阶段包括：头文件展开、宏替换、条件编译、去注释、添加行号

## 写一个标准的宏MIN，这个宏输入两个参数并返回较小一个
- #define MIN(a, b) ((a) < (b) ? (a) : (b))

## 已知数组table，用宏求数组元素个数
- #define COUNT(a) (sizeof(a) / sizeof(a[0]))

## 带参宏和函数的区别
> 带参宏是在编译的预处理阶段被处理的，函数是在运行时进行调用和返回
> 宏替换只占用编译时间，不占用运行时间，而函数调用会占用运行时间
> 带参宏在处理时不分配内存，而函数调用会分配内存
> 宏没有类型限制，而函数有类型限制

## 内联函数的优点和试用场景
1、优点：内联函数和宏定义一样会在原地展开，省去函数调用开销，同时能够完成类型检查
2、缺点： 内联函数的代码会被重复展开，占用更多的内存空间
3、试用场景： 内联函数适用于代码量较小、调用频率较高的场景，例如简单的计算、判断等

## 关键字 volatile 的作用是什么，给出三个不同的例子
1、作用： 告诉编译器不要对该变量进行优化，每次都要从内存中读取最新值
2、试用场景： 当变量的值可能会被其他线程或中断改变时，需要使用 volatile 关键字来确保读取到的是最新值
3、注意事项： 过度使用 volatile 关键字会导致性能下降，因为每次访问变量都需要从内存中读取
4、例子：并行设备的硬件寄存器(状态寄存器)
        中断服务子程序中访问的变量
        多线程应用中被几个线程共享的变量(防止死锁)

## 预处理标识 #pragma once 的作用是什么
> 防止头文件被重复包含
> 当头文件被包含时，会在头文件的开头添加一个标识，防止重复包含
> 当头文件再次被包含时，会检查是否已经添加了标识，如果已经添加了标识，就会直接跳过头文件的内容

## 关键字 static 的作用是什么
> static 修饰局部变量时 ：  改变其储存位置，存储在静态区；改变其生命周期，只在定义它的函数内可见
> static 修饰全局变量时 ：  改变其可见性，只在定义它的文件内可见
> static 修饰函数时 ：  改变其作用域，只在定义它的文件内可见

## 下面的函数会有什么问题
```c
int square(volatile int *ptr)
{
    return *ptr * *ptr;
}
```
该函数的目的是用来返回指针ptr指向值的平方，但是，由于 ptr 指向一个volatile 型参数，编译器将产生以下代码
```c
int square(volatile int *ptr)
{
    int a,b;
    a = *ptr;
    b = *ptr;
    return a * b;
}
```
但是因为此时的 *ptr 值可能被意想不到改变，所以 a和b可能是不同的，这段代码的返回值可能不是所期望的平方值，正确的代码如下
```c
long square(volatile int *ptr)
{
    int a;
    a = *ptr;
    return a * a;
}
```

## sizeof 和 strlen 的区别
> sizeof 是关键字 strlen 是函数
> sizeof 用来计算冠梁、数据类型所占内存的字节数，sizeof(数组名)得到的是数组所占字节数，sizeof(指针)得到的是指针所占字节数
> strlen 用来计算字符串的长度，不包括字符串结束符 \0 ，strlen(指针)得到的是指针指向的字符串的长度 

```c
#include <stdio.h>
#include <string.h>

int main() {
    char str1[] = "Hello"; // 定义一个字符数组，内容为 'H','e','l','l','o','\0'
    char *str2 = "Hello";  // 定义一个指针，指向字符串常量 "Hello"
    
    // 场景 1：sizeof 数组 vs strlen 数组
    printf("sizeof(str1)  = %zu\n", sizeof(str1));   // 输出：6 （整个数组大小：5个字符 + 1个'\0' = 6字节）
    printf("strlen(str1)  = %zu\n", strlen(str1));   // 输出：5 （有效字符个数：H e l l o）
    
    // 场景 2：sizeof 指针 vs strlen 指针
    printf("sizeof(str2)  = %zu\n", sizeof(str2));   // 输出：8 （str2 是一个指针变量，它本身占8字节）
    printf("strlen(str2)  = %zu\n", strlen(str2));   // 输出：5 （顺着指针找字符串长度，找到了 "Hello" 的5个字符）
    
    // 场景 3：对指针使用 sizeof 的常见误区
    char buffer[100] = "Hello";
    char *p = buffer;
    
    printf("sizeof(buffer)= %zu\n", sizeof(buffer)); // 输出：100 （整个数组的大小）
    printf("sizeof(p)     = %zu\n", sizeof(p));      // 输出：8   （指针 p 的大小，不是它指向的内容的大小）
    printf("strlen(p)     = %zu\n", strlen(p));      // 输出：5   （buffer 中字符串的长度）
    
    // 场景 4：没有 \0 的情况
    char chaos[] = {'H', 'e', 'l', 'l', 'o'}; // 没有结尾的 '\0'
    printf("sizeof(chaos) = %zu\n", sizeof(chaos));   // 输出：5 （数组有5个字符，所以大小是5）
    // printf("strlen(chaos) = %zu\n", strlen(chaos)); // 危险！没有 '\0'，strlen 会一直往后数，直到在内存某处碰巧遇到0，结果不可预测
    
    return 0;
}
```

## C 语言编译过程中， 关键字 volatile 和 extern 分别在哪个阶段起作用
> volatile 在编译阶段起作用，extern 在链接阶段起作用

## const 和 #define 的异同
> 异：const 有数据类型，编译器可以做静态类型检查，但是宏定义没有类型，可能会导致类型出错
> 同：两者都可以定义常数
