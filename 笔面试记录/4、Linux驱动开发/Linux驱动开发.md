# 驱动开发
## Linux 驱动程序的功能是什么
1、对设备初始化和释放
2、进行内核和硬件的数据交互
3、检测和处理设备出现的错误

## 内核程序中申请内存使用什么函数
- kmalloc() 、kzalloc()、 vmalloc()

```c
void *kmalloc(size_t size,gfp_t flags);
1、申请连续的物理内存
2、对应的内存释放函数是 void kfree(const void *addr);
3、常用的flag有 GFP_ATOMIC (分配内存是一个原子过程)、GFP_KERNEL(正常分配内存)、GFP_DMA (给DMA控制器分配内存)

void *kzalloc(size_t size, gfp_t flags);
1、相对于 kmalloc() ，kzalloc() 会将申请的内存初始化为0
2、对应的内存释放函数是 void kfree(const void *addr);

void *vmalloc(size_t size);
1、申请非连续的物理内存
2、对应的内存释放函数是 void vfree(const void *addr);
3、常用的flag有 GFP_ATOMIC (分配内存是一个原子过程)、GFP_KERNEL(正常分配内存)、GFP_DMA (给DMA控制器分配内存)

注意： vmalloc()和 vfree() 可以睡眠，不能在中断上下文中使用

```


## 内核程序申请内存和应用程序申请内存有什么区别 
内核程序申请内存空间用的函数是kmalloc() 、kzalloc()、 vmalloc()
应用程序时malloc 

1、kmalloc、kzalloc 直接分配连续的物理地址，虚拟地址也是连续的
2、vmalloc 分配连续的虚拟地址，但是物理地址不一定连续，分配时实际分配了物理内存，不过这个物理内存页面是在公共页表进行映射，并没有在本进程的页表上进行映射，当触发内存时，触发 do_page_fault() 缺页中断才完成页表的同步工作
3、malloc 是用户空间申请内存的方法，分配连续的虚拟地址，物理地址一般不会连续，在分配时并没有做实际物理页的分配动作，实际分配物理页的动作是在 do_page_fault 异常中完成的 

## 自旋锁在和信号量在互斥使用时需要注意什么，在中断服务程序里面的互斥是使用自旋锁还是信号量
1、使用自旋锁的进程不会睡眠，使用信号量的进程会睡眠
2、中断服务程序使用的是自旋锁，原因是中断服务程序处于中断上下文，中断上下文不参与调度，没有保护现场和恢复现场，一旦睡眠就回不来了

## Linux 中引入模块机制有什么好处
1、应用程序在退出时，可以不管资源的释放以及其他的清楚工作，而把这些任务都交给模块退出函数（exit）
2、模块机制有助于缩短模块的开发周期

## Linux 设备驱动程序中，使用那两个函数进行中断处理程序的注册和注销
1、注册中断
int request_irq(unsigned int irq, irq_handler_t handler, unsigned long flags, const char *name, void *dev);
 irq：中断号
 handler：中断处理函数
 flags：中断标志位
 name：中断名称
 dev：设备结构体指针
2、注销中断
void free_irq(unsigned int irq, void *dev_id);
 irq：中断号
 dev_id：设备结构体指针

## 写一个中断服务程序需要注意的地方
1、中断服务程序应该尽可能短，把能放在下半部的任务尽量放在下半部
2、中断服务程序中不能调用可能会睡眠的函数，中断期间是完全占用CPU的，不存在内核调度，中断被阻塞会导致其他进程无法推进
3、中断服务函数的返回值要用操作系统定义的宏，不要用自己的宏
4、中断服务函数不能有不可重入函数，比如打印

## 并发是什么 驱动里面为什么要有互斥控制
1、并发是指多个执行单元同时、并行地被执行，而并发的执行单元对共享资源的访问是互斥的
2、解决竞态问题的途径是保证对共享资源的互斥访问 
3、访问共享资源的代码区域被称为临界区，临界区需要某种互斥机制加以保护，中断屏蔽，原子操作，信号量，自旋锁
