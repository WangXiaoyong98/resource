1、预处理 
    1.1、 预处理阶段做了哪些操作 
        1、对所有 # 开头的语句进行处理，包括包含头文件，定义宏，替换#define 定义的宏
        2、删除注释
        3、添加行号和文件名标识，方便编译器产生的调用或者当编译错误时可以警告显示行号
        4、保留所有的 #pragma 编译指令

    预处理指令是 gcc -E main.c -o main.i

    编译指令是 gcc -S main.i -o main.s

    汇编指令是 gcc -c main.s -o main.o

5、预处理标识符 #error
    #error 预处理指令作用是在编译程序时，只要遇到 #error 就会生成一个编译错误提示信息，并且停止编译，语法格式为 #error error message


6、define 和 const 
    define 是文本替换
    1、内存区域不同 define 常量的生命周期是在编译期，不分配内存，存在与代码段，const 常量存在于数据段，并在堆栈中分配空间，可以用于传递
    2、数据类型不同，define 常量没有数据类型，const 实际存在，并且可以编译器检查


7、#include<filename.h>和#include “filename.h”
#include<filename.h>是从标准库路径开始搜索

#include“filename.h”是先从工作路径开始搜索，再从标准库路径开始搜索

10、静态链接和动态链接的联系与区别
动态库 ：
    1、链接时不复制，程序运行有系统动态加载到内存，只加载一次
    2、加载速度比静态链接慢
    3、发布程序需要提供依赖的动态库
静态库 ： 
    1、静态库被打包在程序中，加载速度快
    2、程序发布无需提供静态库
