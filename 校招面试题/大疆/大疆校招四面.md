2、STM32 处理中断的过程
一、中断处理的宏观流程
一个中断从触发到处理完成，大致会经历以下几个阶段：
中断请求：外设（如定时器、串口、GPIO）产生了中断事件，并将中断信号发送给 NVIC。
中断仲裁：NVIC 根据中断的优先级，判断是否应该响应。如果满足条件（如没有更高优先级的中断正在处理），就会通知 CPU 核心。
中断响应（硬件自动完成）：CPU 核心在完成当前指令后，开始响应中断。这个阶段由硬件自动完成一系列压栈、取向量等操作。
中断处理（软件执行）：CPU 跳转到对应的中断服务程序（ISR，Interrupt Service Routine） 中执行我们编写的代码。
中断返回：执行完 ISR 后，执行特定的中断返回指令，硬件自动恢复之前压栈的现场，CPU 回到被中断的任务继续执行。

8、上下文切换的时候系统做了些什么

保存当前任务的状态：硬件自动保存一部分，软件保存另一部分
选择下一个要运行的任务：调度器根据优先级/策略选出

9、struct 字节对齐
struct 字节对齐是编译器为了提高 CPU 访问内存的效率，而在结构体成员之间插入的"空洞"（padding）
字节对齐是编译器用空间换时间的策略，让每个成员都能在"自然对齐"的地址上快速访问。
常见对齐指令
1. #pragma pack - 修改对齐方式
```C
#pragma pack(1)  // 按 1 字节对齐（紧凑模式）
struct packed {
    char  c;
    int   i;
    short s;
};
#pragma pack()   // 恢复默认对齐

// 此时结构体大小 = 1+4+2 = 7 字节（无 padding）
```
2. __attribute__((aligned(n))) - GCC 指定对齐
```C
// 指定整个结构体按 16 字节对齐
struct __attribute__((aligned(16))) dma_buffer {
    uint8_t data[64];
    uint32_t len;
};

// 指定成员对齐
struct {
    int a;
    int b __attribute__((aligned(64)));  // b 放在 64 字节边界
} foo;
```

12、为什么局部变量未初始化时值不确定？
局部变量分配在栈上。每次函数调用时，栈指针移动，为局部变量腾出空间，但不会清空这块内存。


13、那么嵌入式中栈的工作机制是什么？
栈是一块"后进先出"的内存区域，由SP寄存器指向当前栈顶，每次函数调用、局部变量分配、中断发生时，都会在栈上压入数据；函数返回时再弹出恢复。

14、我连续调用同一个函数两次，他的局部变量初始化结果是否会一致？
对于普通的局部变量，两次调用可能得到相同的结果，也可能不同，取决于两次调用之间栈上的数据是否被修改。对于静态局部变量，则肯定不同。
普通局部变量：取决于栈上的"遗留数据"

变量类型	                两次调用结果是否一致	    原因
普通局部变量（未初始化）	    不一定	                取决于中间是否有函数调用修改栈
普通局部变量（初始化）	        一致（都是初始值）	        每次都重新初始化
静态局部变量	                不一致（递增）	        只初始化一次，值保留

15、静态局部变量与局部变量的区别
局部变量在函数每次调用时创建，结束时销毁，存储在栈上；静态局部变量在程序运行期间只初始化一次，生命周期贯穿整个程序，存储在静态存储区。

16、静态局部变量存储在静态区，那么静态区的创建和消失是在什么时候
静态区在程序加载时创建，在程序退出时销毁。 具体来说：
创建：程序启动时（执行 main 函数之前），由操作系统或运行时环境一次性分配并初始化
销毁：程序正常退出（或 exit 被调用）后，由操作系统回收

1. 操作系统加载程序到内存
2. 分配静态存储区（大小在编译时已确定）
3. 初始化静态区：
   - .data 段：填充初始值（global_var=10, file_static=20, func_static=30）
   - .bss 段：清零（未初始化的全局/静态变量）
4. 调用构造函数（C++ 中全局对象的构造函数）
5. 调用 main 函数  <-- 你写的代码从这里开始执行

19、信号量、互斥量底层是怎么设计的？
信号量和互斥量的底层设计核心是：

原子操作：保证计数值增减的原子性

等待队列：管理等待线程

自旋锁：保护内核对象自身

调度器配合：阻塞和唤醒线程

特殊机制：优先级继承（互斥量）

一句话总结：信号量是带计数的等待队列，互斥量是带所有者和优先级继承的信号量特例。
