8、STM32 中断控制器是什么？
中断管理与使能：NVIC 可以独立地使能（Enable）或除能（Disable）每一个外设中断。你可以精确控制哪些事件可以触发中断，哪些被屏蔽。

优先级管理：这是 NVIC 最核心的功能。它允许你为每个中断设置不同的优先级。当多个中断同时发生时，NVIC 会比较它们的优先级，选择优先级最高的那个通知 CPU 去处理。

中断挂起与激活：如果一个高优先级的中断正在处理，此时来了一个低优先级的中断，NVIC 会把这个低优先级中断标记为挂起（Pending），等待高优先级中断处理完后再去处理它。同时，NVIC 也记录了当前正在被 CPU 处理的中断是哪个，即激活（Active）状态。

中断嵌套控制：NVIC 支持硬件级的中断嵌套。这意味着，当 CPU 正在处理一个低优先级的中断时，如果一个高优先级的中断到来了，NVIC 可以通知 CPU 暂停当前的中断处理，去响应这个更高优先级的请求。处理完高优先级中断后，再自动返回继续处理之前的低优先级中断。这一切都由 NVIC 硬件协调，无需软件干预。

9、外部 RAM 和内部 RAM 区别，怎么分配
内部 RAM 速度快、使用简单，但容量小；外部 RAM 容量大、成本低，但速度慢且需要额外初始化。关键任务的数据放内部，大块不常用的数据放外部。

10、外部总线和内部总线区别

内部总线是 CPU 内核的"私人高速通道"，访问无延迟；外部总线是 CPU 对外的"公共桥梁"，需要协议转换和等待，速度慢但可扩展。

11、MCU 上的固件，数据是怎么分配的
固件在编译链接时由链接脚本决定如何分配，运行时由启动代码负责初始化。主要分为只读区（Flash）、已初始化读写区（RAM）、未初始化区（RAM）和堆栈区（RAM）。

12、MCU 启动流程
MCU 启动流程分为三个阶段：硬件初始化（复位后）、汇编启动代码执行、C 运行时环境建立，最后跳转到 main 函数。

13、IAP 是怎么升级的，突然断电怎么办
：IAP（In-Application Programming）升级是通过Bootloader + App的双区设计实现的。突然断电的防护主要依靠备份机制、标志位保护和数据校验，确保下次启动时能恢复到上一个稳定版本或重新开始升级。
