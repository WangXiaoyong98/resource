```C 
//反转链表

struct ListNode {
    int val;
    struct ListNode* next;
};

struct ListNode* reverseList(struct ListNode* head) {
    struct ListNode *prev = NULL;
    struct ListNode *curr = head;
    struct ListNode *next = NULL;
    
    while (curr != NULL) {
        // 保存下一个节点
        next = curr->next;
        // 反转当前节点的指向
        curr->next = prev;
        // 移动指针
        prev = curr;
        curr = next;
    }
    
    return prev; // prev 是新的头节点
}
```

2、Linux 内核空间与用户空间的区别？
特性	    内核空间	                用户空间
访问权限	完全访问硬件和所有内存	     受限访问，不能直接操作硬件
地址范围	高位地址（如3G+ ~ 4G）	    低位地址（0 ~ 3G）
运行级别	Ring 0（最高特权级）	    Ring 3（最低特权级）
崩溃影响	导致整个系统崩溃	        只影响当前进程
内存访问	可以直接访问物理内存	    只能访问虚拟内存

                    +----------+
                    |  创建    |

饥饿产生的原因： 低优先级任务长时间得不到CPU时间
解决方案：时间片调度

任务优先级反转：指高优先级任务被低优先级任务阻塞，而中等优先级任务抢占CPU的现象。
解决方案：
a. 优先级继承协议
b. 优先级天花板协议

6.1：系统上电阶段 ：
; 系统上电后CPU处于最基本的状态
; 此时内存尚未初始化，只能使用内部SRAM

; CPU复位向量地址（不同架构不同）
; ARM: 0x00000000 或 0xFFFF0000
; x86: 0xFFFFFFF0
; RISC-V: 0x00001000

; 复位后CPU状态：
; - 关中断
; - 设置CPU为管理模式/SVC模式
; - 禁用MMU和Cache
; - 栈指针未定义

7、objdump 如何反汇编 如何查看堆栈段
```bash 
# 反汇编所有代码段
objdump -d program

# 反汇编并显示机器码
objdump -d -S program    # -S 混合显示源代码和汇编

# 反汇编特定函数
objdump -d program | grep -A20 "<main>:"

# 以Intel语法显示（默认是AT&T语法）
objdump -d -M intel program

# 反汇编所有段的内容
objdump -D program    # -D 反汇编所有段
```

8、系统启动会如何加载程序

用户输入命令或点击图标
        ↓
Shell/图形界面解析命令
        ↓
系统调用 execve()
        ↓
内核程序加载器
        ↓
解析ELF可执行文件
        ↓
创建进程地址空间
        ↓
加载程序段到内存
        ↓
动态链接器加载
        ↓
跳转到程序入口
        ↓
程序开始执行

程序加载的关键步骤：

解析ELF文件：读取文件头、程序头

创建地址空间：建立mm_struct和VMA

加载段：将代码和数据映射到内存

加载动态链接器：如果有依赖库

设置栈和堆：为运行时准备

建立页表：使虚拟地址可用

转移控制权：跳转到程序入口
