3、Linux 中进程是什么
进程的核心要素
一个进程在内存中主要由以下几部分组成（结合刚才聊的内存分区）：
代码段：存放程序的机器指令（只读）。
数据段：存放全局变量和静态变量。
堆：用于动态内存分配（malloc 申请的空间）。
栈：存放局部变量、函数参数和返回地址。
内核数据结构：在内核空间中，Linux 为每个进程维护一个叫做 task_struct 的结构体，里面记录了进程的所有信息（后面会详细说）。

Linux 如何管理进程（内核视角）
Linux 内核通过一个叫 task_struct（任务结构体，也叫进程描述符）的数据结构来管理每一个进程。你可以把它想象成进程在操作系统里的身份证档案。
这个结构体包含了极其详细的信息：
PID：进程 ID，唯一标识一个进程。
状态：运行中、休眠中、暂停、僵尸等。
优先级：CPU 调度时决定谁先执行。
程序计数器：记录进程当前执行到哪一行指令了。
内存指针：指向代码段、数据段、堆、栈的位置。
上下文数据：进程暂停时，寄存器的值保存在这里，恢复时才能接着运行。
打开的文件列表：记录该进程打开了哪些文件。
父子进程关系：记录是谁创建了它，它又创建了谁。

进程的生命周期：状态流转
一个进程在运行中不是一成不变的，它会在不同状态间切换：
运行态：进程正在 CPU 上执行，或者等待被调度（就绪）。
可中断睡眠态：进程在等待某个事件完成（如等待用户输入、等待磁盘读取数据）。此时不占用 CPU。可以被信号唤醒。
不可中断睡眠态：同样在等待，但不会响应任何信号。通常是在等待硬件条件，强制杀掉也杀不掉（如等待 IO 完成）。
停止态：进程被暂停了（通常是通过发送 SIGSTOP 信号），可以通过 SIGCONT 让它继续运行。
僵尸态：进程已经终止，但它占用的资源还没有被父进程回收，在内核的进程表中还占着一个位置（留着最后的“遗言”给父进程）。

进程的创建：fork 与 exec
在 Linux 中，创建进程有两个关键的系统调用：
fork()：分身术
创建一个新的进程（子进程）。
子进程几乎是父进程的完整副本（代码、数据、堆、栈都复制一份，但实际物理内存会使用写时拷贝技术优化）。
fork 调用一次，返回两次。在父进程中返回子进程的 PID；在子进程中返回 0。
exec() 系列：变身术
将当前进程的代码段、数据段等全部替换成一个全新的程序（比如在 Shell 里执行 ls）。
执行成功后，原来的程序逻辑就不复存在了，开始执行新程序。

4、多线程编程中的信号量，互斥量，队列
互斥量：解决锁门问题，保护共享资源不被同时修改
信号量：协调多个线程对有限资源的访问或执行顺序
队列：在线程间安全传输数据 

互斥量：
a.线程在访问共享资源前，必须先加锁
b.锁如果被其他线程持有，该线程就会阻塞等待
c.访问结束后，必须解锁

信号量：一个计数器，用来控制同时访问共享资源的线程数量，或用于线程间的顺序同步
两个原子操作
P操作（wait/procure）：信号量值减1，如果结果小于0，线程阻塞
V操作（post/signal）：信号量值加1，如果结果≤0，唤醒一个等待线程

5、SPI 总线协议
同步、全双工、主从式
SPI 的灵活性体现在它有4种通信模式，由时钟极性（CPOL）和时钟相位（CPHA）决定。主设备和从设备必须工作在相同的模式下才能正常通信。
CPOL：决定时钟线在空闲时的电平。
CPOL = 0：空闲时时钟为低电平。
CPOL = 1：空闲时时钟为高电平。
CPHA：决定数据在哪个时钟边沿被采样。
CPHA = 0：在时钟的第一个边沿采样数据。
CPHA = 1：在时钟的第二个边沿采样数据。


6、I2C 协议
它仅需两根线就能挂载多达上百个设备
关键特性：开漏输出 + 上拉电阻
I²C 的两根线都是开漏结构。这意味着设备只能将线拉低（接地），不能主动拉高。必须通过外部上拉电阻将线拉至高电平。
好处：可以避免多个设备同时输出高低电平导致的短路，是实现“线与”功能和总线仲裁的基础
起始条件（S）
动作：当 SCL 为高电平时，SDA 线发生一个高到低的跳变。
意义：通知总线上所有设备，总线被占用，准备开始传输。
地址帧传输
主设备发送7位从设备地址 + 1位读/写位。
0 表示主设备将要写数据到从设备。
1 表示主设备将要读取从设备的数据。
此时，总线上所有从设备都会比对这7位地址。
应答位（ACK）
动作：在第9个时钟周期，发送设备释放SDA线（拉高），接收设备必须将SDA线拉低。
意义：
如果从设备拉低了SDA（产生ACK），表示“地址匹配，我准备好了，请继续”。
如果从设备没有拉低SDA（即NACK，保持高电平），表示“无人响应或无法接收”。
数据帧传输
每发送完一个地址或一个字节数据后，都会跟随着一个应答位（ACK/NACK）。
数据发送的顺序是 MSB first（最高位在前）。
停止条件（P）
动作：当 SCL 为高电平时，SDA 线发生一个低到高的跳变。
意义：通知总线上所有设备，传输结束，总线恢复空闲状态。

8、指针和引用

特性	指针 (Pointer)	引用 (Reference)
本质	是一个变量，存储地址值	是原变量的别名，不占独立存储（语法层面）
初始化	可以延迟初始化，先声明后赋值	必须立即初始化，绑定后不可改变
可修改性	可以改变指向，指向另一个变量	终身绑定，不能改绑到其他变量
内存占用	通常占用固定大小（如 8 字节 64位）	理论上不占空间（实现上可能占，但不可见）
空值	可以有 nullptr/NULL，表示指向空	不能为空，必须引用一个合法对象
解引用	需要用 * 运算符访问内容	直接使用，像普通变量一样
多级	支持多级指针（如 int**）	只有一级引用，没有引用的引用
运算	支持算术运算（p++, p+1）	不支持算术运算
数组	可以有指针数组	可以有数组引用，但不能有引用数组

```C
//初始化要求不同
int a = 10;

// 指针可以只声明不初始化
int* p;       // OK，但值是野指针（危险）
p = &a;       // 后来再赋值

// 引用必须立即初始化
int& r;       // 编译错误！引用必须初始化
int& r2 = a;  // OK
```
```C
int a = 10;

// 指针需要解引用
int* p = &a;
*p = 20;      // 需要 * 来访问

// 引用直接使用
int& r = a;
r = 20;       // 直接像普通变量一样使用
```
指针是一个存储地址的变量，可以改变指向，可以为空，需要解引用才能访问内容；
引用是一个变量的别名，必须初始化，终身绑定，使用时就像普通变量一样。

C语言的三种内存分配方式

1. 静态分配
发生时间：程序编译和链接时决定，程序加载时分配。
存放区域：全局/静态区（数据段和 BSS 段）。
生命周期：从程序开始运行到程序结束。
分配对象：
全局变量：定义在函数外部的变量。
静态变量：使用 static 关键字修饰的变量（包括全局静态和局部静态）。
字符串常量：如 "hello"，存储在常量区（只读）。
特点：
由编译器自动管理，无需程序员手动释放。
未初始化的全局/静态变量会被编译器自动清零（BSS 段）。
在程序的整个执行过程中始终存在。

2. 自动分配
发生时间：在函数被调用时。
存放区域：栈区。
生命周期：从变量定义开始到包含它的代码块（如函数体、复合语句）结束。
分配对象：
局部变量（在函数内部定义的变量，不加 static）。
函数参数。
特点：
自动创建和销毁：进入代码块时分配，退出时释放。
速度快：通过移动栈顶指针实现，效率极高。
大小有限：栈的大小通常较小（几 MB 到几十 MB），递归过深或定义过大的局部数组会导致栈溢出。
内存内容不确定：如果不显式初始化，局部变量的值是随机的（栈上原有的脏数据）。

3. 动态分配
发生时间：程序运行时，由程序员通过特定函数主动申请。
存放区域：堆区。
生命周期：从 malloc / calloc 成功返回开始，到程序员调用 free 释放为止。
分配对象：任何需要在运行时才能确定大小或生命周期的数据（如链表、变长数组、从文件读取的大数据）。
核心函数：
malloc(size_t size)：分配 size 字节的内存，内容不初始化（保留原值）。
calloc(size_t nmemb, size_t size)：分配 nmemb * size 字节的内存，并自动清零。
realloc(void *ptr, size_t new_size)：调整已分配内存的大小。
free(void *ptr)：释放之前动态分配的内存。
特点：
灵活性高：可以分配大块内存，大小受限于系统可用内存。
手动管理：必须由程序员负责释放，否则会导致内存泄漏。
速度较慢：分配算法比栈分配复杂，需要查找空闲内存块。
内存碎片：频繁分配和释放可能导致堆内存碎片化。

总结：
静态分配：为全局和静态变量提供稳定、持久的存储。
自动分配：为函数内部的局部变量提供高效、自动管理的临时存储。
动态分配：提供最大的灵活性，让程序在运行时根据实际需要申请和释放内存。

11、什么是死锁，如何避免死锁
死锁是指：两个或者多个线程在争夺资源时相互等待对方持有资源而陷入一种永久阻塞状态，每个线程都在等待一个永远无法被释放的资源，导致所有相关线程都无法继续执行。

条件	说明
1. 互斥	资源一次只能被一个线程占用（如互斥锁）
2. 持有并等待	线程持有至少一个资源，同时在等待其他资源
3. 不可剥夺	资源只能由持有它的线程主动释放，不能被强制剥夺
4. 循环等待	存在一组线程 {T1, T2, ..., Tn}，T1 在等 T2 的资源，T2 在等 T3 的资源，...，Tn 在等 T1 的资源

串口 异步、全双工的串行通信协议。
只需要两根线就能实现双向通信.
```text

 数据帧格式

停止位：拉高电平（0→1），表示这一帧结束

```

18、现在的 PC 机器用的是大端模式还是小端模式？ARM 开发板呢？大端模式和小端模式的区别？
现在的PC机：几乎全是小端模式。无论是你用的Windows笔记本，还是Linux台式机，只要是基于Intel或AMD的x86/x86_64架构，都采用小端。

ARM开发板：绝大多数情况也是小端模式。ARM架构本身是双端的，即可配置为大端或小端，但在实际应用中，为了和PC生态兼容，几乎所有主流的操作系统（如Linux、Android、iOS）和编译器，都将ARM设备默认设置为并运行在小端模式。

大端和小端，指的是多字节数据在内存中存放的字节顺序。

想象你要存放一个4位数的密码 1234：

1 是最高位（千位，类似数据的高位字节）

4 是最低位（个位，类似数据的低位字节）

大端模式 (Big-Endian)：像我们写数字一样，把高位写在左边。它把数据的高位字节（1）存放在内存的低地址（第一个盒子）。
小端模式 (Little-Endian)：反着来，把数据的低位字节（4）存放在内存的低地址（第一个盒子）。
